# Session Notes

This folder contains development session notes for the CRM for Athletic Directors project.

## Naming Convention

Session note files follow this format:
```
YYYY-MM-DD_Brief-Description-Of-Work.md
```

**Examples:**
- `2026-01-17_Expandable-People-Menu-Implementation.md`
- `2026-01-20_Database-Schema-Updates.md`
- `2026-01-22_School-Management-Features.md`

## What to Include in Session Notes

Each session note should document:
1. **Overview** - Brief summary of what was accomplished
2. **Changes Made** - Detailed list of modifications
3. **Files Modified** - Which files were changed and why
4. **Git Details** - Commit hash, branch, repository info
5. **Next Steps** - Future improvements or follow-up tasks
6. **Useful Commands** - Relevant commands for this session's work

## How to Use

### Starting a New Session
1. Reference previous session notes to understand recent work
2. Tell Claude: "Check the latest session notes in /session-notes folder"

### Ending a Session
1. Ask Claude to create a new session note
2. Use descriptive title that captures the main work done
3. Include date in YYYY-MM-DD format

### Reviewing Past Work
- Browse by date to find specific sessions
- Use file names to quickly identify topics
- Git commit hashes link session notes to code changes

## Tips
- Keep notes concise but comprehensive
- Include screenshots or diagrams if helpful
- Link to relevant documentation or resources
- Note any blockers or issues encountered
